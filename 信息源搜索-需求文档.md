# 信息源搜索 · 需求文档（移动端）

> 本文档面向设计与开发，描述「信息源搜索」在移动端的功能目标、业务逻辑与交互逻辑。
> 不含数据结构定义与视觉设计细节。

## 0. 概述

「信息源搜索」是内容中心的核心检索能力，帮助用户从信息源中检索事项、查看事项详情、并基于所选事项进入创作。整体分为三大功能点：

1. **信息搜索（首页）**：选择搜索范围、输入关键词发起检索，并浏览每日事件。
2. **搜索结果列表页**：展示 AI 总结与结果事项列表，支持多选并发布帖子。
3. **事项详情页**：查看单条事项的完整内容，并回溯其被检索到的推理路径。

三处的「事项」为同一概念：事项分为**总结事项**（对一篇文章的归纳，下挂若干子事项）与**子事项**（文章片段的归纳，可脱离总结事项被单独引用/展示）。事项详情页在三个入口共用同一套内容结构。

## 0.1 全局导航

- 底部一级菜单保留「首页 / 消息 / 我的」三个主入口。
- 一级菜单栏仅展示图标，不展示文字标签。
- 当前激活页通过图标高亮表达状态。

## 1. 信息搜索（首页）

### 1.1 功能目标

为用户提供统一的检索入口：选择搜索范围、输入关键词发起搜索；同时以「每日事件」的形式提供可直接浏览与进入详情的内容，降低冷启动的输入成本。

### 1.2 业务逻辑

**包含字段：**

- **搜索关键词**：用户在搜索框输入的检索词。
- **推荐问题**：一组示例问题，用于搜索框占位文案的轮播展示（可有多条）。
- **搜索范围**：
    - 范围类型：公共信息源 / 个人信息源（二选一）。
    - 个人信息源来源分类：我的信息源（用户自建）/ 与我共享（他人共享给我的信息源，**仅企业版提供**）。
    - 个人信息源清单：按来源分类展示，支持跨分类多选。
    - 已勾选信息源数量（跨分类合计）。
- **每日事件（列表）**：
    - 当前日期：当前展示的事件所属日期，可通过日历选择切换。
    - 每条事件字段：发生时间、标题、摘要、来源（来源名称 + 来源头像）、相关子事项数量、展开/收起状态。
    - 子事项字段：标题、摘要。
- **往期事件（日期选择）**：以日历形式选择要查看的历史日期；含月份切换、当前日期与所选日期的标识。

**默认顺序：**

- 每日事件顶层条目按**事件发生时间倒序**（最新在上）。
- 总结事项展开后的子事项按**事项序号正序**（1、2、3…）。

**默认值：**

- 搜索范围默认为「公共信息源」。
- 搜索框占位文案默认展示推荐问题中的第一条，并在输入为空时按固定间隔轮播下一条。
- 每日事件默认展示当天日期。

**状态定义与处理：**

- **搜索范围 — 公共信息源（默认）**：默认包含全部公共信息，用户无需再选择具体信息源，可直接搜索。
- **搜索范围 — 个人信息源**：需要用户从个人信息源清单中勾选至少一个作为搜索范围；未勾选任何信息源时不可发起搜索，并给出提示。
- **个人信息源 — 来源分类切换**：在「我的信息源」「与我共享」两个分类间切换，分别展示对应清单；某分类无数据时展示占位文案。「与我共享」为企业版特性。
- **日期选择 — 可选范围**：仅可选择当天及过去的日期；未来日期不可选，且不可翻至未来月份。
- **每日事件 — 有数据**：渲染事件列表。
- **每日事件 — 无数据**：渲染占位文案，不出现事件卡。
- **子事项 — 收起（默认）/ 展开**：每个总结事项独立维护展开/收起状态，互不影响。
- **搜索关键词 — 为空 / 非空**：为空时占位文案继续轮播；提交时若关键词为空，则以当前占位展示的推荐问题作为检索词。

### 1.3 交互逻辑

- **选择搜索范围**：点击搜索范围入口，调起搜索范围选择面板：
    - 选择「公共信息源」：面板仅需确认，范围回显为「公共信息」。
    - 选择「个人信息源」：面板中出现「我的信息源 / 与我共享」两个来源分类，切换后展示对应清单；用户可跨分类勾选/取消多个信息源，实时回显已勾选数量；确认后范围回显为「个人信息 · N 个」。「与我共享」仅企业版提供。
    - 若选择个人信息源但未勾选任何信息源，确认时提示需至少选择一个信息源，不关闭面板。
- **输入与发起搜索**：
    - 输入框为空时，占位文案按固定间隔轮播推荐问题。
    - 点击提交或在输入框回车发起搜索；关键词为空时以当前占位展示的推荐问题作为检索词。
    - 若当前范围为个人信息源且未勾选信息源，提示先选择信息源，不进入结果页。
    - 发起成功后进入「搜索结果列表页」。
- **浏览每日事件**：
    - 有数据态：纵向展示事件列表，总结事项默认收起，仅展示标题、摘要、来源与「N 个相关子事项」入口。
    - 点击「N 个相关子事项」入口：仅切换该总结事项的展开/收起，展开后在其下追加渲染全部子事项，不进入详情。
    - 点击事件卡主体或任一子事项：进入「事项详情页」，展示该事项内容。
    - 无数据态：展示占位文案。
- **选择往期日期**：点击「往期事件」入口，调起日历面板：
    - 日历按月展示，可切换上/下月，但**不可翻至未来月份**。
    - 标识当前日期与当前所选日期；未来日期置灰不可选。
    - 选择某日期后，回填每日事件的当前日期，并刷新为该日期的事件列表。

### 1.4 全局搜索导流

- 顶部搜索图标可打开全局搜索页，作为信息源搜索的辅助导流入口。
- 全局搜索页的发现区标题固定为「每日事件」。
- 发现区右侧提供「信息源搜索 >」快捷入口，点击后进入信息源搜索首页。
- 发现区不单独展示日期副标题，时间信息保留在事件时间轴与卡片内容中承载。

## 2. 搜索结果列表页

### 2.1 功能目标

在一次检索后，向用户提供由检索结果凝练的 AI 总结与可操作的结果事项列表；支持用户多选事项并据此发起创作，形成「搜索 → 选取 → 创作」的完整链路。

### 2.2 业务逻辑

**包含字段：**

- **当前搜索关键词**：本次检索使用的关键词，可在结果页顶部编辑后重新检索。
- **AI 总结**：
    - 总结正文：对本次检索结果的归纳文字。
    - 引用标注：正文中对所参考结果事项的引用序号。
    - 参考数据条数：总结所参考的结果事项数量。
    - 展开/收起状态。
- **排序方式**：结果列表的排序方式，在「智能排序」与「时间排序」之间切换（默认智能排序）。
- **多选开关状态**：是否处于多选态。
- **结果事项（列表）**：
    - 每条结果字段：序号、标题、摘要、来源（来源名称 + 来源头像）、发生时间、相关子事项数量、选中状态。
    - 子事项字段：标题、摘要。
- **已选事项数量**与**可选上限**：单次最多可选 100 个事项。
- **发布配置**：
    - 发布助手：可选的发布助手列表，默认选中上一次使用的助手。
    - 内容类型：根据已选事项数量自动决定（无需用户选择）。
    - 内容字数上限：生成内容的字数上限（1000 字）。

**默认顺序：**

- 结果事项默认按**与关键词的相似度倒序**（智能排序，最相关在上）；切换为「时间排序」时按**事件发生时间倒序**（最新在上）。
- 总结事项展开后的子事项按**事项序号正序**。

**默认值：**

- 排序方式默认为「智能排序」，每次重新发起搜索时重置为智能排序。
- 进入结果页时默认非多选态，已选事项为空。
- AI 总结默认收起。
- 发布助手默认选中上一次使用的助手。

**内容类型自动配置规则（按已选事项数量）：**

- 已选 1–5 个：帖子。
- 已选 6–30 个：主题。
- 已选 31 个及以上：总结。

**状态定义与处理：**

- **结果列表 — 有结果 / 无结果**：无结果时展示占位文案，不出现结果卡。
- **排序方式 — 智能排序（默认）/ 时间排序**：切换后按对应规则重排结果列表；重新检索时重置为智能排序。
- **AI 总结 — 收起（默认）/ 展开**：收起时截断展示，展开时展示完整正文。
- **多选 — 关闭（默认）/ 开启**：
    - 关闭态：点击结果卡进入事项详情；不展示勾选框与底部已选栏。
    - 开启态：结果卡展示勾选框，点击结果卡切换选中；底部展示已选栏。
    - 由开启切换回关闭时，清空已选事项。
- **选择数量 — 已达上限**：已选达到 100 时，继续勾选给出上限提示，不再增加。
- **发布可用性 — 已选为 0 / ≥1**：已选为 0 时「发布帖子」不可用；已选 ≥1 时可用。
- **子事项 — 收起（默认）/ 展开**：与首页一致，独立维护。

### 2.3 交互逻辑

- **顶部搜索**：顶部展示可编辑的搜索框（含清除入口）；编辑后回车即以新关键词重新检索并刷新本页内容；点击返回回到上一级页面，而不是固定跳转信息源搜索首页。
- **AI 总结**：
    - 收起态：展示截断后的总结正文与「展开」入口，底部展示「AI 总结 · 参考以下前 N 个数据」。
    - 展开态：展示完整总结正文与「收起」入口。
    - 正文中的引用标注用于指示该结论参考了列表中的第几条结果。
- **排序与多选控件**：
    - 「排序切换」：在「智能排序」（相似度倒序）与「时间排序」（发生时间倒序）之间切换，控件文案与图标随当前排序方式变化，切换后列表实时重排。
    - 「多选」：切换多选态开启/关闭。
    - 「信息源」入口：在结果页 tab 栏最右侧提供带星标 icon 的文字按钮「信息源」，点击后进入信息源搜索首页，并沿用当前搜索词继续检索。
- **结果列表**：
    - 非多选态：点击结果卡进入「事项详情页」；点击「N 个相关子事项」仅展开/收起子事项，点击子事项进入其详情。
    - 多选态：结果卡出现勾选框，点击结果卡切换选中；已选数量与上限在底部已选栏实时回显；达到上限继续勾选给出提示。
- **已选栏（多选态）**：展示「已选 N / 100」、「清空」、「发布帖子」。
    - 清空：取消全部选中。
    - 发布帖子：已选 ≥1 时可用，调起发布面板。
- **发布帖子面板**：
    - 展示可选发布助手，默认选中上次使用；用户可切换。
    - 展示由已选数量自动决定的内容类型，以及内容字数上限，供用户知晓（不可编辑）。
    - 「取消」：关闭面板，返回结果页，保留已选。
    - 「开始创作」：提示创作已开始、完成后将通知用户；随后清空已选并退出结果页。

## 3. 事项详情页

### 3.1 功能目标

在「每日事件」「搜索结果」等入口下，统一查看单条事项（总结事项或子事项）的完整内容；对由检索得到的事项，额外提供「推理路径」以回溯其被检索到的过程。

### 3.2 业务逻辑

**包含字段：**

- **事项标题**。
- **属性-值标签**：一组结构化标签，按事项性质组合展示。
- **AI 摘要**：对该事项的归纳文字；内容超长时支持折叠（展开/收起）。
- **原文参考**：
    - 原文参考正文：子事项使用其自身原文；总结事项按其下子事项的默认顺序聚合各子事项的原文；内容超长时支持折叠（展开/收起）。
    - 查看链接入口：跳转至该事项的原始来源页。
- **推理路径**（仅「搜索结果」入口进入的事项具备）：
    - 原始问题：本次检索使用的关键词。
    - 检索命中：该事项被命中的检索方式与命中来源。
    - 最终结果：最终命中并展示的事项。

**状态定义与处理：**

- **事项来源 — 搜索结果事项 / 非搜索事项**：
    - 搜索结果事项：具备推理路径，详情内展示「推理路径」入口。
    - 非搜索事项（每日事件、报告来源、消息来源等）：不具备推理路径，不展示该入口。
- **推理路径面板 — 关闭（默认）/ 打开**：详情页内默认仅展示一个紧凑入口（卡片内小标签）；打开后以独立面板展示完整推理路径。
- **长文本折叠（AI 摘要 / 原文参考）**：
    - AI 摘要超出 2 行时默认按 **2 行省略**，并提供「展开/收起」；未超出 2 行时不出现该入口。
    - 原文参考超出展示区域时默认截断，并保留底部渐隐提示与「展开/收起」；内容未超出时不出现该入口。
- **字段缺失处理**：任一字段缺失时，不渲染其对应区块（不留空占位）。
- **详情为模态叠加**：详情独立于其调出来源；关闭后不改变调出来源页的任何状态。

### 3.3 交互逻辑

- **打开态**：依次展示 事项标题 → 属性-值标签 → AI 摘要 →（推理路径入口，若有）→ 原文参考（含查看链接入口）。缺失的字段不展示对应区块。
- **推理路径**：
    - 详情内仅展示一个紧凑入口（卡片内小标签），避免占据正文空间（便于优先阅读可能较长的原文参考）。
    - 点击入口：调起推理路径面板，依次展示 原始问题 → 检索命中 → 最终结果。
    - 关闭面板：回到事项详情，详情内容与状态不变。
    - 非搜索来源的事项不展示该入口。
- **长文本折叠**：
    - AI 摘要：超出 2 行时默认两行省略，点击「展开」查看全文、点击「收起」还原。
    - 原文参考：内容超长时默认截断并保留底部渐隐提示，点击「展开」查看全文、点击「收起」还原。
    - 两类内容在未超出各自展示阈值时均不显示该入口。
- **原文参考**：展示原文参考正文；点击「查看链接」跳转至该事项的原始来源页。
- **关闭详情**：点击返回或关闭，回到调出该详情的来源（首页每日事件 / 搜索结果列表页），来源页状态保持不变。
